COMPREHENSIVE PROJECT SPECIFICATION FOR REPLIT AI
================================================

PROJECT NAME: PromptCraft AI
OBJECTIVE: Build a complete, professional full-stack web application that helps users create, optimize, and manage AI prompts for different content types (images, videos, websites, code, text, audio, presentations).

========================================
CORE CONCEPT
========================================

Create an intelligent prompt engineering assistant that connects to multiple AI APIs (ChatGPT, Gemini, Claude) and uses best practices from each platform to generate optimized, detailed prompts. The app should function as an expert prompt engineer - similar to how ChatGPT helps create prompts, but specialized and optimized for various platforms and content types.

========================================
TECHNICAL STACK
========================================

Frontend:
- React 18+ with TypeScript
- Tailwind CSS for styling
- shadcn/ui component library
- React Router for navigation
- Framer Motion for animations
- Lucide React for icons

Backend:
- Node.js with Express
- Replit Database (key-value store)
- Replit Auth for authentication

AI APIs:
- OpenAI GPT-4/GPT-4o API
- Google Gemini API
- Anthropic Claude API

Voice Input:
- Web Speech API (browser native)
- OpenAI Whisper API (fallback)

SECTION 3: PLATFORM-SPECIFIC BEST PRACTICES

**OPENAI / CHATGPT:**
Based on official documentation:
- Be specific and descriptive
- Use examples for desired output format (few-shot)
- System prompts for context
- Temperature: lower for precision, higher for creativity
- Structured outputs with JSON mode
- "Think step by step" for reasoning tasks
- Provide positive instructions (what to do, not what not to do)
- Use delimiters (""", ```, ###) for clarity
- Split complex tasks into subtasks
- Give the model time to "think"
- Test and iterate

**GOOGLE GEMINI:**
Based on Gemini documentation:
- Be precise and direct
- Use consistent structure (XML or Markdown)
- Define all ambiguous parameters
- Control output length explicitly
- Place important context first
- Use anchor phrases to reference previous content
- Leverage thinking capabilities for complex tasks
- Temperature default: 1.0 (adjust based on needs)
- Multimodal: combine text with images effectively
- Avoid persuasive language in prompts
- Structured output with JSON schema

**ANTHROPIC CLAUDE:**
Based on Claude documentation:
- Extreme clarity and specificity
- Assign clear roles/personas
- Use XML tags for structure (<instructions>, <context>, <examples>)
- Prefill for format control
- Few-shot prompting with examples
- "Think step by step" for complex reasoning
- Constitutional AI: include ethical guidelines
- Long context: leverage 200k token window
- Avoid vague or open-ended instructions
- Positive framing

**CURSOR AI:**
Based on Cursor best practices:
- Minimal, focused context
- Use @ to reference files
- Notepads for complex feature specifications
- .cursorrules for persistent guidelines
- Close irrelevant tabs/files
- Test-driven development approach
- Iterate with specific feedback
- Enable debugging/logging
- One feature at a time

**REPLIT AI:**
Based on Replit Agent best practices:
- Break down into small, incremental steps
- Specify tech stack clearly
- "Build incrementally" approach
- Provide clear context upfront
- Use positive instructions
- Iterate and refine continuously
- Use checkpoints for versioning
- Leverage native integrations
